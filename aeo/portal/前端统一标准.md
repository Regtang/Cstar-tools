# 喜事达工具平台 · 前端统一标准 v1.0

所有挂在 **bot.cstar.com** 的工具，部署前必须满足本标准。目的是让同事各自开发的工具在配色、风格、基本功能上保持一致，给客户和内部一个统一、专业的体验。

设计基准来自门户网站与「Cstar 装箱软件」。

---

## 二、统一样式表 cstar-ui.css（必须引入）

不要自己从零写配色和组件。工具 HTML 的 `<head>` 引入共享样式表即可自动统一：

```html
<link rel="stylesheet" href="/cstar-ui.css">
```

`<body>` 加 `class="cstar-ui"` 套用统一字体与背景。常用组件类（均以 `cs-` 前缀，不会和你已有样式冲突）：

- 布局：`cs-wrap`、`cs-grid-2`（左输入右结果）、`cs-row`（两列）
- 页头：`cs-header`（青色渐变标题条）
- 卡片：`cs-card` + `cs-card-body`
- 表单：`cs-label`、`cs-input`、`cs-select`、`cs-textarea`
- 按钮：`cs-btn` + `cs-btn-primary` / `cs-btn-accent` / `cs-btn-soft` / `cs-btn-ghost`（可加 `cs-btn-block`）
- 徽章：`cs-badge` + `cs-badge-ok/warn/danger/info`
- 提示条：`cs-note` + `cs-note-info/warn/danger`
- 表格：`cs-table`；校验错误：`cs-field-error` + `cs-error-text`

线上地址：https://bot.cstar.com/cstar-ui.css

## 三、嵌入模式（必须支持）

首页会用弹窗以 `?embed=1` 内嵌你的工具，此时要隐藏工具自带的大页头，避免「双标题」。在 `<head>` 加：

```html
<script>
  document.documentElement.classList.toggle("embed-mode",
    new URLSearchParams(location.search).get("embed") === "1");
</script>
```

cstar-ui.css 已内置 `html.embed-mode .cs-header { display:none }`，用 `cs-header` 即自动生效。

## 四·五、数据存储：本地 vs 后端

引入标准功能库 `cstar-tool.js`（`<script src="/cstar-tool.js"></script>`），全局对象 `Cstar` 提供两层存储：

**本地存储 `Cstar.store`（localStorage）** —— 免登录、离线可用，但只在当前这台设备、这个浏览器里，换设备/清缓存即丢失，多人不共享。用于表单自动暂存、离线兜底。

**后端存储 `Cstar.cloud`（服务端 SQLite，按账号 + 工具隔离）** —— 需登录（喜事达通行证），数据存到服务器，跨设备同步、可后台统计；服务端强制只能读写"自己 + 本工具"的数据。所有方法异步、自动带登录令牌，遇未登录默认引导到通行证页。

两种用法（`slug` 为本工具英文标识，如 `duty-calc`）：

```js
// KV：单份配置 / 表单暂存
await Cstar.cloud.save('duty-calc', 'form', {hs:'8471', qty:10});
const f = await Cstar.cloud.load('duty-calc', 'form', {});   // 未登录/无数据返回默认值
await Cstar.cloud.remove('duty-calc', 'form');

// 台账：一条条记录（保存计算结果、提交历史等）
const {id} = await Cstar.cloud.add('duty-calc', {no:'D001', total:1234});
const list = await Cstar.cloud.list('duty-calc');            // [{id,data,updated_at}]
const hit  = await Cstar.cloud.list('duty-calc', '8471');    // 关键词检索
await Cstar.cloud.update('duty-calc', id, {no:'D001', total:2000});
await Cstar.cloud.del('duty-calc', id);
await Cstar.cloud.exportExcel('duty-calc', [['no','单号'],['total','金额']]);  // 拉全量导出
```

建议：本地 `store` 做随手自动暂存，`cloud` 做"保存到云端 / 我的台账"。凡是产生"一条条记录"、或需要跨设备/多人/后台统计的工具，都应改用 `Cstar.cloud`。

## 五、部署前两步（每个工具上线前必做）

### 第 1 步：前端统一
- [ ] 引入 `/cstar-ui.css`，`body` 加 `cstar-ui`
- [ ] 主色为青 `#007892`，强调用红 `#e54c5e`，无蓝/紫等非品牌主色
- [ ] 页头用 `cs-header`，并支持 `?embed=1` 嵌入模式（无双标题）
- [ ] 字体、圆角、按钮、卡片与门户/装箱软件观感一致

### 第 2 步：功能测试 + 基本功能基线（参照装箱软件）
- [ ] **示例/演示数据**：一键填入样例，方便试用与演示
- [ ] **输入校验 + 错误提示**：必填项、格式、范围校验，错误友好提示
- [ ] **导出/打印**：结果可导出（CSV/TXT）或打印
- [ ] **响应式 + 本地保存**：手机可用；输入用 localStorage 自动暂存不丢失
- [ ] **需跨设备/多人/统计的数据**：改用后端存储 `Cstar.cloud`（见四·五），不要只存 localStorage
- [ ] 功能跑通：核心流程、边界值、异常输入都测一遍

两步都打勾后，再交给我部署到 bot.cstar.com（我会复核并加首页入口卡片）。

## 六、部署方式（由平台维护者执行）

1. 工具 HTML 放到站点根目录 → `https://bot.cstar.com/你的工具.html`
2. 在门户首页 `portal/index.html` 的 `TOOLS` 数组加一项（`embed:'/你的工具.html?embed=1'`）
3. 浏览器验证：配色统一、弹窗无双标题、功能基线可用
