# 喜事达AEO认证管理平台 —— 企业服务器上线手册

本手册面向 IT/运维，把"上线"变成复制粘贴。提供 **Docker** 和 **纯 Linux(systemd)** 两条路径，二选一即可；HTTPS 为可选增强。文末有上线检查清单。

> 说明：应用本身已通过 16 项自动化测试与生产多进程启动验证。`deploy/.env.production` 已内置一份随机密钥，可直接用，也可自行重新生成。

---

## 0. 准备

- 一台服务器（Linux 推荐，2 核 2G 起步即可，SQLite 单机足够中小企业量级）。
- 把整个 `xishida-aeo` 目录上传到服务器，例如 `/opt/xishida-aeo`。
- 决定访问方式：内网 `IP:8000` 直连，或域名 + HTTPS（见第 3 节）。

---

## 路径 A：Docker 部署（最省心，推荐）

### A1. 安装 Docker（若已装跳过）
```bash
curl -fsSL https://get.docker.com | sh
```

### A2. 配置环境变量
```bash
cd /opt/xishida-aeo
cp deploy/.env.production .env
vi .env          # 确认/修改 AEO_SECRET_KEY 与 AEO_ADMIN_PASSWORD
```

### A3. 启动
```bash
docker compose up -d        # 首次会自动构建镜像
docker compose ps           # 状态应为 healthy
curl http://127.0.0.1:8000/api/health   # {"status":"ok"}
```

### A4. 访问
浏览器打开 `http://服务器IP:8000`，用 admin + 你在 .env 设置的密码登录。

### 升级 / 维护
```bash
docker compose down         # 停（数据保留在卷 aeo-data）
docker compose up -d --build  # 更新代码后重建
docker compose logs -f      # 看日志
```

## 路径 B：纯 Linux + systemd（不用 Docker）

### B1. 安装 Python 与依赖
```bash
sudo apt update && sudo apt install -y python3 python3-venv python3-pip
sudo useradd -r -s /usr/sbin/nologin aeo        # 专用运行账号
sudo mkdir -p /opt/xishida-aeo /var/lib/xishida-aeo
sudo cp -r ./* /opt/xishida-aeo/
cd /opt/xishida-aeo
sudo python3 -m venv .venv
sudo .venv/bin/pip install -r requirements.txt
sudo chown -R aeo:aeo /opt/xishida-aeo /var/lib/xishida-aeo
```

### B2. 安装服务
```bash
sudo cp deploy/xishida-aeo.service /etc/systemd/system/
sudo vi /etc/systemd/system/xishida-aeo.service   # 改 AEO_SECRET_KEY / AEO_ADMIN_PASSWORD
sudo systemctl daemon-reload
sudo systemctl enable --now xishida-aeo
sudo systemctl status xishida-aeo                 # active (running)
curl http://127.0.0.1:8000/api/health
```

### 维护
```bash
journalctl -u xishida-aeo -f        # 实时日志
sudo systemctl restart xishida-aeo  # 重启
```

> 注：systemd 模板里应用监听 `127.0.0.1:8000`（只对本机），配合第 3 节 Nginx 对外。
> 若要内网直连不配 Nginx，把 ExecStart 的 `--host 127.0.0.1` 改为 `--host 0.0.0.0`，并放行防火墙 8000 端口。

## 3. 配置域名 + HTTPS（可选，对外发布强烈建议）

### 3.1 装 Nginx 并申请证书
```bash
sudo apt install -y nginx certbot python3-certbot-nginx
sudo cp deploy/nginx.conf /etc/nginx/conf.d/xishida-aeo.conf
sudo vi /etc/nginx/conf.d/xishida-aeo.conf     # 把 aeo.your-company.com 改成你的域名
sudo certbot --nginx -d aeo.your-company.com   # 自动签发并配置证书
sudo nginx -t && sudo systemctl reload nginx
```

### 3.2 防火墙
```bash
sudo ufw allow 80,443/tcp     # 只放行 Web 端口
# 不要对公网放行 8000（应用端口仅本机/内网）
```

之后通过 `https://aeo.your-company.com` 访问。

## 4. 上线后第一件事（安全加固）

1. 用 admin 登录 → 「用户管理」给每位使用者建独立账号、分配角色。
2. 「设置」里修改 admin 密码（即便已用环境变量，也建议再改一次）。
3. 删除/停用不需要的演示账号（guanwu/neishen/caiwu/wuliu/anbao）。
4. 「设置 → 清空业务数据」，清掉示例数据，开始录入企业真实数据。

## 5. 数据备份（合规要求：留存≥3年）

- **Docker**：备份数据卷
  ```bash
  docker run --rm -v xishida-aeo_aeo-data:/data -v $(pwd):/backup alpine \
    tar czf /backup/aeo-backup-$(date +%F).tar.gz -C /data .
  ```
- **systemd**：直接备份目录 `/var/lib/xishida-aeo`（含 `aeo.db`）。
- 建议加 cron 每日备份并异地保存。

## 7. 性能与扩展

- SQLite 适合中小企业（数万~数十万条记录）。并发很高时升级 PostgreSQL：改 `backend/database.py` 连接串为 `postgresql+psycopg://user:pwd@host/db`，装 `psycopg[binary]`，模型层无需改动。
- 横向扩展多副本时，所有副本必须共享同一 `AEO_SECRET_KEY`，并改用 PostgreSQL 作为共享数据库。

---

## 8. 常见问题

- **健康检查不过 / 起不来**：看日志（`docker compose logs` 或 `journalctl -u xishida-aeo`），多为端口被占或依赖未装。
- **登录提示密码错**：`AEO_ADMIN_PASSWORD` 只在"首次建库"生效；库已存在时请用系统内现有密码，或删库重建（会清空数据）。
- **页面打开但功能异常**：确认是通过后端地址访问（前端需要后端 API），不要直接双击 index.html。
