## 一、平台概况

- 线上地址：**https://bot.regs.com**
- 服务器：腾讯云香港轻量 **ubuntu@43.129.179.78**（Ubuntu，Docker + nginx 反向代理）
- 架构：FastAPI + SQLite + Docker compose；nginx 路由 `/`（首页/工具静态）、`/aeo/`（AEO 平台）、`/api/`（后端）、`/tools/`、`/packer/`（装箱软件）
- 数据库：SQLite 持久化在 Docker 卷 `/data`（含用户、AEO 标准、工具注册表等）

## 二、本地代码与运维位置（均在桌面）

- `~/Desktop/xishida-aeo/`：**主代码**
  - `backend/`：FastAPI（main.py、seed.py、models.py、auth.py、database.py）
  - `frontend/`：AEO 平台 SPA（index.html、app.js、styles.css、version.js、VERSION）

## 三、部署 / 同步方式（重要）

1. 改完 `~/Desktop/xishida-aeo/` 里的代码后，**双击 `~/Desktop/喜事达运维/部署.command`** 上线（约 30 秒，工具数应为 9）。
2. 把改动同步到仓库：把改的文件 `cp` 到 `~/Desktop/cstar-tools/` 对应位置，**双击 `同步GitHub.command`** 推送。
3. **已知问题**：国内→GitHub 的 443 端口偶发 SSL 超时，推送会失败；**本地提交不会丢，多双击一两次即可补推**。同理香港服务器→www.cstar.com 也偶发超时（已加重试）。
4. 缓存：index.html 已设 no-cache，资源带 ?v=版本号；用户首次看旧版只需 **Cmd+Shift+R** 强刷一次。

  - 守法规范扩展到更多关联主体
- 标准框架在 `seed.py` 的 `STANDARD_FRAME`，启动时版本守护迁移（检测旧版自动刷新，状态重置为待评估）。

- 登录：演示管理员 `admin / admin123`；平台管理员 `pmadmin`（角色 platform，与 AEO 隔离）；管理后台 `/admin.html`。

## 六、HS 速查全量数据接入（最新完成）

- 数据源：喜事达自家 API **`POST https://www.cstar.com/findRateDeclByPage`**（用户提供，文档在 `uploads/findRateDeclByPage-api*.md`）。返回 goodsCode/goodsName/declNotes(申报要素，**用「|」分隔**)/各档税率/customsSupervision/inspectionDesc/goodsDesc 等。
- 后端代理：`main.py` 的 **`POST /api/hs/search`**（公开，无需登录），转发到上游，规避跨域；含重试（2 次/12 秒）；地址可用环境变量 `HS_RATE_API` 覆盖。
- 前端：HS 工具点「查」/回车 → 在线查全量；输入时本地常用库即时联想；全量无匹配（口语词如"锂电池"，官方名"锂离子蓄电池"）或超时 → 自动回退本地库并提示。
- 已实测通过：查"牛肉"返回 18 条权威结果，详情含 16 项申报要素、税率、监管 4ABEFx 等。
